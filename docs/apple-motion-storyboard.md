# Chroni Apple Motion Storyboard

## 1. Hero

- Initial: headline, download CTA, course notice, idle mascot, and a restrained Daily Planner window.
- Scroll 0-15%: notice settles into the desktop stage; mascot looks active; bubble says to drop the notice.
- Scroll 15-32%: notice moves toward the mascot; deadline, deliverables, method, and risk are highlighted.
- Scroll 32-52%: the structured extraction panel moves from the notice toward the product window.
- Scroll 52-73%: the real Daily Planner window expands; the planning decision becomes readable.
- Scroll 73-100%: mascot changes to reminder state and gives the 19:00 start prompt.
- End: notice evidence, structured result, mascot, real product UI, value statement, and CTA remain in one poster-like frame.
- Static/mobile/reduced: the final complete frame is shown without scroll mapping.
- Assets: real Daily Planner screenshot and idle/study/response Tongluv frames.

## 2. Input to extraction

- Initial: clean stage with no sources.
- Early scroll: course chat, PDF, email, and screenshot enter as distinct source objects.
- Middle: source density rises and the study mascot activates.
- Late: source objects recede while evidence chips move toward the result.
- End: three structured tasks retain source type, deadline, and deliverables.
- Static/mobile: sources and final result appear in normal vertical order.
- Assets: study mascot frame; source objects are accessible HTML representations of supported inputs.

## 3. Extraction to plan

- Initial: date, fixed meeting, and empty day capacity are visible.
- Entry: task-pool items retain identity while moving into the timeline.
- Order: database assignment, README, and final test settle according to deadline and risk.
- End: healthy status, capacity totals, and the Agent's scheduling reason are visible.
- Static/reduced: all blocks appear at final positions.
- Asset: complete Daily Planner screenshot follows the build scene as product proof.

## 4. Mascot states

- Stage: one central mascot, system events on the left, response and relevant UI on the right.
- States: idle, reading, planning, reminder, done, and rest.
- Interaction: selecting an event replaces the mascot in the same spatial position and updates the response preview.
- Keyboard: arrow keys move between states.
- Reduced: the state changes by an immediate short crossfade with no ambient breathing.
- Assets: idle, study, wake, play, and sleep Tongluv frames.

## 5. Daily plan

- Fixed event occupies 14:00-16:00 before Agent tasks.
- Database assignment takes the complete 19:00 focus period.
- README and final testing share the later period with distinct color and width.
- Time block size communicates duration; color communicates task identity and risk.

## 6. Journal

- Initial: three completed timeline records remain separate.
- Reveal: records visually converge behind a dated journal page.
- End: completion score, focus time, deferred work, and tomorrow's count are readable.
- Static/mobile: both records and journal remain visible with no transform dependency.

## 7. Final CTA

- Motion intensity is reduced after privacy and FAQ.
- The wake mascot, direct value statement, and platform-matched download button form the last decision point.
- GitHub Release remains the direct installer source.
