# Chroni 小红书发布与录屏计划

## 一句话定位

我做了一个会读 DDL 的桌面小伙伴。

展开表达：把课程通知、截图或文件交给 Chroni，它会提取截止事项、拆成步骤、安排今天的时间块，再由桌宠提醒下一步。

## 核心传播原则

- 先展示结果，再解释 Agent 架构。
- 只展示已经实现的能力，不宣传官方免费模型额度。
- 录屏使用 `examples/demo/`，不出现真实姓名、群聊、学校或 API Key。
- 同一条视频只讲一个闭环：拖入 -> 识别 -> 安排今天 -> 桌宠提醒。
- 系统签名警告和模型费用必须在下载说明中如实披露。

## 15 秒视频脚本

| 时间 | 画面 | 字幕 |
| --- | --- | --- |
| 0-2 秒 | 长课程通知快速滚动 | DDL 又藏在通知里？ |
| 2-4 秒 | 把示例 TXT 拖到桌宠 | 直接交给桌宠 |
| 4-6 秒 | 桌宠阅读动作 | 本地读取 / OCR |
| 6-9 秒 | 展示识别出的标题、截止时间和提交物 | 自动整理 DDL |
| 9-12 秒 | 点击“帮我安排今天” | 不只提醒，还会拆成今天能做的计划 |
| 12-15 秒 | 每日时间轴和桌宠气泡 | Chroni，一个会读 DDL 的桌面伙伴 |

## 30 秒视频脚本

1. 开头 0-4 秒：“如果你也经常忘 DDL，可以试试这个桌宠。”
2. 4-8 秒：展示课程群长通知，然后拖入 Chroni。
3. 8-12 秒：桌宠进入阅读动作，气泡显示正在处理。
4. 12-17 秒：展示提取出的任务、截止时间、提交物和提交方式。
5. 17-22 秒：打开 TaskPlan，快速扫过步骤、依赖和耗时。
6. 22-26 秒：点击“帮我安排今天”，生成时间轴。
7. 26-30 秒：桌宠提醒下一步，字幕：“开源内测中，Windows/macOS 可下载。”

## 60 秒深度脚本

1. 问题：课程通知太长，传统待办需要手工复制标题和日期。
2. 输入：支持文字、PDF、DOCX、XLSX、ICS、图片和扫描 PDF。
3. 抽取：先本地解析/OCR，再由本地规则或用户配置模型理解语义。
4. 校验：不确定的时间会要求确认，不会默默编造日程。
5. 规划：DDL 进入 TaskPlan，拆成可编辑步骤。
6. 执行：Agent 根据容量、依赖和风险安排今天。
7. 交互：桌宠接收拖放、反馈处理状态、打开日程和提醒下一步。
8. 隐私：默认 local-first；启用模型后文本会发送到用户配置的服务。
9. 结尾：展示 GitHub Release 和安装 FAQ。

## 录屏前准备

- 使用新建的测试系统账户或清理过的演示数据目录。
- 关闭聊天、邮件、系统通知预览和密码管理器弹窗。
- API Key 留空或完全打码，不录制粘贴过程。
- 使用 `examples/demo/` 中固定材料。
- Windows 和 macOS 分别录制，画面中标注平台。
- 统一显示缩放和窗口位置，桌宠不要遮住关键按钮。
- 录制 1920x1080 横版母版，再裁切 1080x1440 或 1080x1920 竖版。
- 保留鼠标轨迹，但避免高速晃动。

## 发布正文模板

```text
我做了一个会读 DDL 的桌面小伙伴 Chroni。

把课程通知、文字、PDF、表格或截图拖给它，它会整理截止时间和提交物，再把任务拆成今天能执行的时间块。桌宠不是装饰，它会反馈读取状态、打开日程并提醒下一步。

目前是开源内测版本，支持 Windows 和 macOS。不开模型也能处理结构明确的 DDL；复杂材料可以配置自己的 OpenAI-compatible API Key。

安装包可能触发系统的未签名提示，请只从项目 GitHub Releases 下载并核对 SHA-256。项目地址和安装 FAQ 放在评论区/主页。
```

## 评论区 FAQ

| 问题 | 简短回答 |
| --- | --- |
| 免费吗？ | Chroni 开源；本地规则无需模型费用。自带模型 Key 时按服务商计费。 |
| 不填 Key 能用吗？ | 能处理时间表达明确的基础 DDL，复杂语义建议配置自己的 Key。 |
| 支持什么文件？ | TXT、Markdown、PDF、DOCX、XLSX、ICS 和常见图片等。 |
| 数据会上云吗？ | 默认本地保存；启用模型后，解析文本会发送到用户配置的模型服务。 |
| Windows 为什么提示风险？ | 当前包可能未签名，只从 GitHub Releases 下载并核对校验和。 |
| 有手机版吗？ | 当前专注 Windows/macOS 桌面端。 |
