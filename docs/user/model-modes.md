# Chroni 模型使用方式

Chroni 提供三种真实可用的工作方式。它们共用同一套本地文件解析、来源证据校验、状态存储和失败回退。

| 方式 | 是否联网调用模型 | 凭据 | 适用场景 |
| --- | --- | --- | --- |
| 本地规则 | 否 | 不需要 | 标题、日期、时间较明确的通知 |
| Chroni 内测 | 是 | 可撤销的内测访问码 | 受邀测试者低门槛体验完整抽取与 Agent 规划 |
| 自定义 API | 是 | 用户自己的 API Key | 开发者、长期使用或需要自行控制服务商 |

## 本地规则

没有启用 LLM、没有填写凭据，或模型请求失败时，Chroni 会使用本地规则继续处理。PDF、DOCX、XLSX 和图片仍会先在本机解析或 OCR，不会因为没有模型而变成不可用的演示界面。

本地规则适合结构明确的输入：

```text
请在明天 20:00 前提交数据库作业 PDF，通过课程平台上传。
```

只有“尽快完成”“下次课前交”等模糊表达时，Chroni 才会要求确认，而不会编造日期。

## Chroni 内测

内测模式不要求用户申请 DeepSeek Key。维护者将 DeepSeek 主密钥保存在 Zeabur 环境变量中，给每位或每组测试者发放独立访问码。桌面端只保存访问码，并调用：

```text
https://api-getchroni.zeabur.app/v1/chat/completions
```

网关固定使用允许的 DeepSeek 模型，覆盖客户端提交的模型和思考模式，并执行请求体限制、并发限制、分钟与每日额度、上游超时和脱敏日志。访问码可以单独撤销，不需要更换 DeepSeek 主密钥。

### 安装版配置

1. 从托盘打开控制中心，进入“偏好”。
2. 展开“高级 -> 智能模型服务”。
3. 选择“Chroni 内测”。
4. 填写邀请中提供的访问码。
5. 点击“保存并测试”，成功后开启“启用 LLM 抽取”。

访问码会在系统支持时通过 Electron `safeStorage` 加密保存。应用快照、本地 HTTP API 和界面回填都不会返回明文访问码。

## 自定义 API

选择“自定义 API”后，可填写 OpenAI-compatible Chat Completions 服务的 Base URL、模型 ID 和 API Key。DeepSeek 当前配置为：

```dotenv
CHRONI_LLM_ENABLED=1
CHRONI_LLM_MODE=custom
CHRONI_LLM_BASE_URL=https://api.deepseek.com
CHRONI_LLM_MODEL=deepseek-v4-flash
CHRONI_LLM_API_KEY=replace-with-your-own-key
```

源码开发者可复制根目录 `.env.example` 为 `.env`。优先级为：系统环境变量 > 项目 `.env` > 控制中心保存值。检测到环境变量时，界面不会回填凭据。

## 什么会发送

文件解析和 OCR 始终先在本机进行。启用任一联网模型模式后，Chroni 会发送完成当前抽取或规划所需的解析文本、任务元数据、来源摘要和已筛选的结构化偏好。长文档可能分块覆盖全文，二进制原文件不会作为文件直接上传。

内测网关只记录请求 ID、访问码的匿名标识、状态码、耗时和 token 用量，不记录原始提示词、文件文本、访问码或 DeepSeek Key。处理敏感材料前仍应阅读所用模型服务的隐私政策。

## 常见连接结果

| 提示 | 优先检查 |
| --- | --- |
| 内测访问码无效或已失效 | 访问码是否完整、维护者是否已轮换或撤销 |
| API Key 无效或无权限 | 自定义 Key 是否完整、账户是否开通 API |
| 模型名称不可用 | 自定义模式使用服务商当前模型 ID |
| 请求过于频繁或今日额度已用完 | 等待限流窗口，或联系维护者调整内测额度 |
| 连接超时 | 网络、代理、防火墙和 Base URL |
| 返回内容无法识别 | 重试；仍失败时保留本地规则结果并提交脱敏反馈 |

网关部署、环境变量和访问码轮换见[内测 LLM 网关运维指南](../llm-gateway.md)。
