# Chroni 模型使用方式

## 当前可用的两种方式

Chroni `v0.1.4` 实际提供本地规则和用户自带 API Key 两种方式。它们共用同一套本地证据校验、状态存储和失败回退。

| 方式 | 是否联网调用模型 | 适用内容 | 费用 |
| --- | --- | --- | --- |
| 本地规则 | 否 | 标题、日期、时间较明确的通知 | 无模型调用费用 |
| 用户自带 Key | 是 | 长文、跨段要求、OCR 后复杂语义、增强 TaskPlan | 由模型服务商计费 |

当前没有 Chroni 官方试用额度或官方 Relay。文档中提到的 `official_trial` 是后续架构规划，不是已上线功能。

## 本地规则模式

以下任一情况会使用本地规则：

- 没有填写 API Key。
- “启用 LLM 抽取”处于关闭状态。
- 模型请求失败、超时或返回非法结构，系统自动回退。

本地规则不会把材料发送到模型服务。PDF、DOCX、XLSX 和图片仍会先在本机解析或 OCR。建议输入包含明确动作和完整时间，例如：

```text
请在明天 20:00 前提交数据库作业 PDF，通过课程平台上传。
```

只有“尽快完成”“下次课前交”等模糊表达时，Chroni 可能要求确认，而不会编造日期。

## 用户自带 API Key

### 安装版配置

1. 从托盘打开控制中心。
2. 进入“偏好 -> 高级 -> 大模型 API”。
3. 填写 Base URL、模型 ID 和 API Key。
4. 点击“保存并测试”。
5. 测试成功后开启“启用 LLM 抽取”。
6. 若希望 Agent 规划也使用模型，在 Agent 的规划偏好中单独开启相应选项。

DeepSeek 当前 OpenAI-compatible Base URL 为 `https://api.deepseek.com`。模型 ID 与价格可能变化，请以[DeepSeek 官方 API 文档](https://api-docs.deepseek.com/)为准。

### 源码运行配置

源码开发者可以复制根目录 `.env.example` 为 `.env`。真实 Key 只放在本机 `.env` 或系统环境变量，不得提交到 Git。

```dotenv
CHRONI_LLM_ENABLED=1
CHRONI_LLM_BASE_URL=https://api.deepseek.com
CHRONI_LLM_MODEL=deepseek-v4-flash
CHRONI_LLM_API_KEY=replace-with-your-own-key
```

优先级为：系统环境变量 > 项目 `.env` > 控制中心保存值。检测到环境变量时，界面不会回填 Key。

## Key 如何保存

- 控制中心输入的 Key 在系统支持时使用 Electron `safeStorage` 加密。
- 普通快照和本地 HTTP API 不返回 Key。
- 如果系统安全存储不可用，界面 Key 只在本次运行有效，运行状态会给出提示。
- `.env` 是明文开发配置，必须保持在 Git 忽略范围内。

## 什么会发送给模型

启用模型后，Chroni 会把本地解析或 OCR 得到的文本，以及完成当前抽取或规划所需的结构化上下文发送到你配置的服务。二进制原文件不会作为文件直接上传，但文件中的文字内容可能被发送。

发送敏感材料前，应阅读模型服务商隐私政策。无需模型时关闭“启用 LLM 抽取”，并关闭 Agent 大模型规划。

## 常见连接结果

| 提示 | 优先检查 |
| --- | --- |
| API Key 无效或无权限 | Key 是否完整、账户是否开通 API |
| 模型名称不可用 | 使用服务商当前模型 ID，不使用聊天网页产品名 |
| 额度不足或服务繁忙 | 账户余额、限流、稍后重试 |
| 连接超时 | 网络、代理、防火墙和 Base URL |
| 返回内容无法识别 | 换用兼容 Chat Completions 的模型，或使用本地规则 |

## 后续官方试用的安全边界

未来若提供官方试用，必须经由 Chroni Relay 调用模型，客户端只能持有匿名设备令牌，不能包含官方模型 Key。上线前还需要真实额度、限流、隐私政策、删除机制、监控和成本保护。在这些条件满足前，产品不会显示虚假的剩余额度或免费次数。
