# Chroni 3 分钟快速开始

本指南面向第一次使用 Chroni 的普通用户。无需安装 Node.js，也不要求先填写 API Key。

## 第 1 步：下载

只从 [Chroni Latest Release](https://github.com/miracle121388-a11y/chroni/releases/latest) 下载。

| 你的设备 | 推荐文件 |
| --- | --- |
| Windows 10/11 x64 | `Chroni-<version>-win-x64-setup.exe` |
| 不希望安装的 Windows | `Chroni-<version>-win-x64-portable.exe` |
| Intel 或 Apple Silicon Mac | `Chroni-<version>-mac-universal.dmg` |

遇到 SmartScreen、Gatekeeper 或不知道选哪个文件，请先看[安装 FAQ](./install-faq.md)。

## 第 2 步：启动并认识三个入口

启动后你会看到蓝色桌宠。Chroni 关闭控制中心后仍会在托盘或菜单栏运行。

- 左键桌宠：打开或收起日程抽屉。
- 右键桌宠：打开桌宠菜单。
- 控制中心“每日任务”：查看今天真正要执行的时间块。
- 控制中心“日程”：输入、上传、核对和管理 DDL。
- 控制中心“Agent”：生成或更新今日安排。

## 第 3 步：不用 Key 完成第一次体验

1. 下载仓库中的[课程作业示例](../../examples/demo/01-course-assignment.txt)，或在控制中心输入：

   ```text
   请在明天 20:00 前提交数据分析课程报告 PDF，通过课程平台作业区上传。
   ```

2. 把 TXT 拖到桌宠，或在“日程”的文件区选择“预览抽取”。
3. 桌宠进入阅读动作。先核对标题、截止时间、提交物和提交方式。
4. 预览不会保存；确认后使用“直接填入”，或再次拖入材料进入日程。
5. 如果出现“待确认”，只补充真正缺失的信息。明确的标题和时间不会要求重复填写。
6. 打开该 DDL 的“规划详情”，检查步骤和耗时，选择“确认并启用”。
7. 进入“Agent”，点击“帮我安排今天”。
8. 回到“每日任务”，查看时间轴并完成一个工作块。

这条路径使用本地规则，不会调用模型服务。示例中的“明天”按电脑当前时区计算；如果临近午夜，请核对最终日期。

## 可选：连接智能模型

复杂通知、跨段要求和图片 OCR 文本更适合使用大模型增强。

1. 打开“偏好 -> 高级 -> 智能模型服务”。
2. 受邀内测者选择“Chroni 内测”，填写邀请中提供的访问码。
3. 自带 DeepSeek Key 的用户选择“自定义 API”，Base URL 填写 `https://api.deepseek.com`，模型填写 `deepseek-v4-flash`。
4. 点击“保存并测试”。
5. 测试成功后开启“启用 LLM 抽取”。

详细边界见[模型使用方式](./model-modes.md)和[隐私说明](./privacy.md)。

## 日常使用建议

1. 新材料先预览，复杂或高风险事项再确认后填入。
2. 每天开始工作时运行一次 Agent，生成今日时间块。
3. 拖动时间块调整安排，完成后及时勾选。
4. 截止日期或要求变化时更新来源并重新识别，不要创建多个含义相同的任务。
5. 在“运行状态”检查本地解析、OCR、模型和存储状态。
6. 定期点击“检查更新”，并从 GitHub 发布页核对新版本。

## 退出、数据和反馈

- 关闭控制中心不等于退出。需要彻底退出时，请使用系统托盘或菜单栏中的“退出 Chroni”。
- “运行状态 -> 打开本地数据位置”可以找到本机状态和备份。
- 不要在公开 Issue 中上传 API Key、真实课程通知、私人文件或未脱敏路径。
- 出现问题时按[故障排查](./troubleshooting.md)收集安全信息，再通过[帮助与反馈](./feedback.md)提交。
