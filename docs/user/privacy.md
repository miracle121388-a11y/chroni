# Chroni 隐私说明

本说明描述 Chroni `v0.1.4` 的默认数据边界，不替代你所使用的模型服务商隐私政策。

## 结论先行

- Chroni 是 local-first 桌面应用，日程、来源、TaskPlan、每日任务、偏好和 Agent 结构化记录默认保存在本机。
- 不配置 API Key 或关闭 LLM 时，Chroni 不会调用模型服务。
- 启用用户自带 API Key 后，本地解析或 OCR 得到的文本可能发送到用户配置的模型服务。
- Chroni 不会把真实模型 API Key 写入仓库、README、安装包资源或普通状态快照。
- 当前没有 Chroni 官方 Relay、账户、云同步或官方试用额度。

## 数据流

```text
用户文件/文字
-> 本机文件解析或 OCR
-> 本机证据、日期和字段校验
-> 可选：发送解析文本到用户配置的模型服务
-> 本机保存 DDL、计划、来源和执行状态
```

## 只保存在本机的数据

默认保存在 Electron 用户数据目录：

- DDL、完成状态、重要性和稍后提醒。
- 输入来源与解析后的文字。
- 待确认草稿和用户回答。
- TaskPlan、计划版本、步骤、依赖和预计耗时。
- 每日任务、颜色、时间块、子任务和完成记录。
- 提醒、工作时段、桌宠位置和界面偏好。
- Behavior Memory 中经过用户明确保存的结构化偏好。
- Agent 的结构化摘要、规划来源和工具执行结果。
- 自动备份、OCR 缓存和本地导出文件。

在控制中心选择“运行状态 -> 打开本地数据位置”可以查看实际目录。

## 可能发送到模型服务的数据

只有启用 LLM 抽取或 Agent 大模型规划时，才会发送：

- 本地解析或 OCR 得到的文本片段。
- 来源名称、来源类型和时间上下文。
- 完成当前任务抽取所需的提示词和结构约束。
- 规划所需的任务标题、截止时间、交付物、步骤和偏好摘要。

Chroni 不会把二进制原文件作为文件直接上传，但原文件中解析出的文字可能被发送。模型服务如何保存、训练或处理内容，取决于用户选择的服务商。

## API Key

- 控制中心中的 API Key 在操作系统支持时使用 Electron `safeStorage` 加密后保存。
- `chroni-state.json` 不保存明文 Key，公开 snapshot 也会移除 Key。
- 系统安全存储不可用时，界面填写的 Key 只在当前运行期间有效，并在运行状态中提示。
- 开发用 `.env` 是本机明文文件，已被 `.gitignore` 忽略。不要上传、截图或发送给他人。

## 本地 HTTP API

- 默认只监听 `127.0.0.1`，不对局域网或互联网开放。
- 每次启动生成会话令牌，除健康检查外均要求 Bearer 鉴权。
- 发现文件 `chroni-api.json` 在应用退出后删除。
- 不要把本地令牌交给不可信脚本，也不要随意配置跨域来源。

## 日志与反馈

公开反馈前必须移除：

- API Key、Bearer Token 和 `.env` 内容。
- 真实课程通知、邮件、聊天记录和文件正文。
- 姓名、学号、手机号、邮箱、群号和企业内部信息。
- 完整本地路径和包含用户名的截图。
- 未脱敏的 `chroni-state.json`。

建议只提供 Chroni 版本、操作系统、是否启用 LLM、错误分类、最小复现步骤和使用虚构内容制作的截图。

## 关闭模型调用

1. 打开“偏好 -> 高级”。
2. 关闭“启用 LLM 抽取”。
3. 在 Agent 规划偏好中关闭大模型规划。
4. 若使用环境变量启动，删除或关闭 `CHRONI_LLM_ENABLED`，然后重启。

环境变量优先于界面。如果开关显示“由环境变量控制”，必须在启动环境中关闭。

## 删除本地数据

1. 从托盘或菜单栏完全退出 Chroni。
2. 在退出前通过“运行状态 -> 打开本地数据位置”确认目录。
3. 备份需要保留的导出文件。
4. 删除 Chroni 用户数据目录。
5. 如需彻底移除，再卸载应用或删除 Portable 程序。

删除前请确认没有另一个 Chroni 实例运行。该操作不可撤销。

## 安全问题

API Key 泄露、任意代码执行、鉴权绕过、恶意文件解析、更新劫持或用户数据泄露，请使用 [GitHub Private vulnerability reporting](https://github.com/miracle121388-a11y/chroni/security/advisories/new)，不要创建公开 Issue。
