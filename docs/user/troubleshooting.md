# Chroni 故障排查

先记录 Chroni 版本、操作系统和最小复现步骤，再按错误发生阶段排查。不要把 API Key 或真实材料粘贴到公开反馈中。

## 应用无法启动

1. 确认下载的是当前平台产物，不是 `blockmap`、`latest.yml` 或校验文件。
2. 只从 GitHub Releases 下载，并核对 SHA-256。
3. Windows 检查任务管理器中是否有旧 Chroni 或 Portable 实例。
4. macOS 检查菜单栏和活动监视器。
5. 完全退出后重新启动，不要同时运行 Setup 与 Portable。
6. 若系统提示签名或公证问题，按[安装 FAQ](./install-faq.md)处理，不要关闭系统全局安全功能。

## 启动后只看到桌宠

这是正常的桌面形态。左键桌宠打开日程抽屉，右键打开菜单，再选择“打开控制中心”。也可以检查系统托盘或 macOS 菜单栏。

## 桌宠或窗口不在可见区域

- 重新启动 Chroni，窗口位置会根据当前显示器工作区恢复并校正。
- 临时恢复刚刚断开的显示器，再把桌宠拖回主屏。
- 检查系统缩放比例是否刚发生变化。
- 仍无法恢复时，退出应用并备份用户数据，再在反馈中说明显示器数量、分辨率和缩放比例。

## 文件显示为空或无法读取

检查：

- 文件不是 0 字节。
- 扩展名与真实内容一致，二进制文件不能只改名为 `.txt`。
- 文件没有被其他程序独占或放在失效的网络位置。
- 单个文档不超过 18 MiB，纯文本不超过 2 MiB。
- TXT 使用 UTF-8、UTF-16、GBK 或 GB18030。
- 加密 PDF、旧版 `.doc`、宏工作簿和受保护表格可能无法解析。

先在“日程”中使用“预览抽取”。预览不会保存，错误信息更适合定位问题。

## 图片或扫描 PDF 没有识别结果

- 提高图片清晰度，避免阴影、透视、过小字号和复杂背景。
- 裁掉与通知无关的大面积区域。
- 扫描 PDF 首次处理会初始化 OCR 资源，可能比纯文本慢。
- OCR 结果置信度不足时，Chroni 会拒绝静默生成任务。
- 可以先用系统 OCR 复制文字，再把文字交给 Chroni。

## 明确通知仍然要求补充标题或截止时间

1. 查看抽取预览中的来源文字是否完整。
2. 确认日期和时间包含在同一任务语义中。
3. 避免只写“下次课前”“尽快”“到时候交”等无法唯一确定的表达。
4. 在来源记录中修正文案后重新识别。
5. 使用 `examples/demo/` 的材料对比，确认基础链路正常。

## DeepSeek 连接失败

| 现象 | 处理 |
| --- | --- |
| 鉴权失败 | 重新创建 Key，确认没有空格并具备 API 权限 |
| 模型不存在 | 使用 DeepSeek 官方文档当前列出的模型 ID |
| 额度不足/限流 | 检查余额和调用限制，稍后重试 |
| 超时 | 检查网络、代理、防火墙和 Base URL |
| 空内容或非法 JSON | 换用兼容 Chat Completions 的模型，或关闭 LLM 使用本地规则 |

Base URL 应填写 API 地址，不是 DeepSeek 聊天网页地址。当前官方 OpenAI-compatible 地址为 `https://api.deepseek.com`。

## 界面关闭后程序仍在运行

这是托盘常驻设计。关闭控制中心不会停止提醒、Agent 或桌宠。需要完全退出时，从托盘或菜单栏选择“退出 Chroni”。开发环境可以在启动终端按 `Ctrl+C`。

## 本地 API 端口被占用

默认端口为 `8765`。先退出所有 Chroni 和 Electron 开发实例。开发者可设置 `CHRONI_API_PORT`，安装版普通用户一般不需要修改。

## 数据显示只读、恢复或重置

打开“运行状态”，阅读“本地数据”提示，再点击“打开本地数据位置”。Chroni 会尽量保留损坏原件并使用 `.backup` 自动恢复。

不要直接覆盖仍在运行中的 `chroni-state.json`。先退出应用，复制整个目录作为备份，再处理权限、磁盘空间或损坏文件。

## 自动更新失败

- 当前签名条件可能限制某些平台上的自动更新。
- 打开“运行状态 -> 查看 GitHub 发布页”，手动下载最新版。
- 手动安装前退出 Chroni。
- 安装版覆盖安装通常保留用户数据；仍建议先备份本地数据目录。

## 源码运行出现 `ERR_PNPM_RECURSIVE_RUN_FIRST_FAIL`

这通常只是 pnpm 汇总行，真实错误在它上方第一条 `[renderer]` 或 `[electron]` 输出。

```powershell
npx pnpm@11.7.0 install
npx pnpm@11.7.0 run dev
```

如果 Electron 被当成 Node 运行：

```powershell
Remove-Item Env:ELECTRON_RUN_AS_NODE -ErrorAction SilentlyContinue
```

然后重新启动终端。

## 安全地提交反馈

按照[帮助与反馈](./feedback.md)准备版本、平台、最小步骤和脱敏截图。不要直接上传用户数据目录。安全漏洞使用私密报告渠道。
